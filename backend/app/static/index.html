Verified AI Research Writer

API Reference — v1.0.0

Base URL: http://localhost:8000/api/v1

⚡ Health 1

GET /health
Check if the API server is running. Returns status and version.
no auth no project required

📋 Projects 4

POST /api/v1/projects
Create a new content project. Provide topic, title, tone, content type, SEO keywords, and points to cover. Returns the created project with a UUID.
request body: ProjectCreate
GET /api/v1/projects
List all projects ordered by creation date (newest first). Supports ?skip=0&limit=100 pagination.
query: skip, limit
GET /api/v1/projects/{project_id}
Get a single project by its UUID. Returns full project details including outline, status, and metadata.
path: project_id (UUID)
DELETE /api/v1/projects/{project_id}
Delete a project and all its associated content, claims, evidence, and sources.
path: project_id (UUID) returns 204

✍️ Content 3

POST /api/v1/projects/{project_id}/content/generate
Core pipeline endpoint. Triggers the full multi-agent content generation pipeline: topic planning → web research → evidence retrieval → claim extraction → fact verification → content writing → self-verification. Returns generated markdown, citations, claims, and a verification summary.
path: project_id (UUID) no request body pipeline runs async
GET /api/v1/projects/{project_id}/content
Get all generated content versions for a project, ordered newest first.
path: project_id (UUID)
GET /api/v1/projects/{project_id}/content/latest
Get the most recently generated content for a project. Returns markdown, citations, SEO metadata, and confidence score.
path: project_id (UUID)

🔎 Evidence 2

GET /api/v1/projects/{project_id}/evidence
Get all evidence items collected during research. Each item includes a snippet, source URL, domain, relevance score, and source trust score. Returns average relevance across all evidence.
path: project_id (UUID)
GET /api/v1/projects/{project_id}/evidence/claims/{claim_id}
Get evidence items specifically linked to a single claim. Useful for drilling into what sources support a particular claim.
path: project_id, claim_id (UUID)

✅ Verification 3

GET /api/v1/projects/{project_id}/verification/claims
Get all extracted claims with their verification status, confidence scores, and explanations. Returns aggregate counts: total, verified, unverified, contradicted, unsupported.
path: project_id (UUID)
GET /api/v1/projects/{project_id}/verification/sources
Get source trust metrics. Lists every source URL with its domain, title, and computed trust score. Includes average trust score across all sources.
path: project_id (UUID)
GET /api/v1/projects/{project_id}/verification/dashboard
Full dashboard endpoint. Returns a comprehensive snapshot: project metadata, all claims with statuses and confidence, source trust scores, evidence counts with relevance, and content metrics (word count, overall confidence, citation count).
path: project_id (UUID)
⚙️ Pipeline Flow

When you call POST /api/v1/projects/{id}/content/generate, the system runs 5 stages in sequence:

1. Topic Planner — generates a structured outline with sections and research queries
2. Research — searches web sources (Tavily API or mock), stores evidence, computes trust scores
3. Verification — extracts factual claims from evidence, cross-references sources, assigns confidence scores
4. Content Writer — writes markdown using only verified claims (>0.70 confidence), adds inline citations
5. Self-Verification — audits for hallucination risks, unsupported claims, citation mismatches

Each stage updates the project status so you can poll GET /api/v1/projects/{id} for progress.
